## 一、产品概述

执照认证系统是一套面向服务人员（学员）的线上职业技能认证平台，提供从账号注册、信息认证、在线考试到执照获取的完整闭环流程。系统分为**考生端（移动端 H5）**和**管理端（PC 后台）**两端，管理员可在后台管理考生信息、审核认证材料、追踪考试进度。

---

## 二、目标用户

| 角色 | 描述 |
|------|------|
| 学员（考生） | 需要通过技能认证获取执照的服务人员，在移动端操作 |
| 培训师 | 负责培训学员，查看学员考试进度（后续规划） |
| 校区负责人 / 管理员 | 在 PC 后台管理考生、审核认证材料、查看考试数据 |

---


## 四、功能需求 — 考生端（H5）

### 4.1 账号注册 & 登录

#### 4.1.1 登录鉴权

- 访问任意内部页面前，先校验登录态；未登录则跳转至登录页。
- 登录成功后跳转回触发登录的原页面。

#### 4.1.2 微信授权登录

| 字段 | 说明 |
|------|------|
| 功能 | 接收微信授权，记录授权信息 |
| 账号创建规则 | 以手机号为唯一标识；手机号未注册则自动创建账号 |
| 扩展性 | 账号中心需预先设计多平台支持（微信、好慷在家等） |
| 页面元素 | 图标 + 文案"执照认证平台" |

#### 4.1.3 手机验证码登录 / 注册

**图形验证码**

- 展示4位随机数字图片，点击可刷新。
- 图形验证码通过后，"获取短信验证码"按钮才可点击。

**短信验证码**

- 输入验证码后，提交按钮激活。
- 点击提交逻辑：
  - 协议未勾选 → 弹窗提示"请确认勾选协议"，用户点击"确认"后自动勾选。
  - 验证通过 → 登录成功，缓存登录态，跳转原页面。
  - 验证失败 → 提示"验证异常，请重试"。

**用户协议**

- 底部勾选框"我已阅读并同意《用户服务协议》"。
- 协议文字可点击，点击后弹窗展示完整协议内容。

### 4.3 个人中心

#### 4.3.1 账号信息

| 字段 | 说明 |
|------|------|
| 头像 | 可点击上传，支持本地图片预览 |
| 昵称 | 可编辑文本 |
| 手机号 | 展示脱敏号码（如 138****1234），不可编辑 |
| 个人照片 | 上传后调用 AI 人像验证接口 |

**个人照片 AI 人像验证流程：**

```
上传图片
    ↓
调用 AI 人像验证接口
    ↓
验证中：显示加载动画 + "AI人像验证中..."
    ↓
验证成功 → 绿色提示"验证通过"
验证失败 → 红色提示具体原因（如"未检测到人脸"/"检测到多人"/"照片模糊"）
          → 显示"重新上传"按钮
```

> **备注**：个人照片后续用于监考场景人脸识别，本期先完成上传与 AI 验证，不拓展至监考模块。

#### 4.3.2 证件信息

支持身份证、健康证的上传与管理。

**身份证：**

| 要素 | 说明 |
|------|------|
| 上传 | 正、反面各一张 |
| OCR 识别 | 上传后自动识别姓名、证件号、有效期 |
| 交叉校验 | OCR 识别的姓名 / 证件号与账号信息不一致时，弹窗警告"证件信息与账号不符" |
| 有效期提示 | 有效期已过时，展示红色标签"已过期" |

**健康证：**

| 要素 | 说明 |
|------|------|
| 上传 | 单张图片 |
| OCR 识别 | 自动识别有效期 |
| 有效期内 | 绿色标签"有效" |
| 已过期 | 红色标签"已过期，请重新上传" |

> 任何时候均可重新上传证件信息，覆盖旧数据。

#### 4.3.3 认证材料

考生上传认证材料，由管理员在后台审核。

| 材料类型 | 上传方 | 审核方 | 状态流转 |
|----------|--------|--------|----------|
| 体检报告 | 考生 | 管理员 | 待审核 → 通过 / 不通过 |
| 背调材料 | 考生 | 管理员 | 待审核 → 通过 / 不通过 |

**各状态展示：**

| 状态 | 展示 |
|------|------|
| 未上传 | 上传按钮 |
| 待审核 | 黄色标签"待审核" + 预览缩略图 |
| 通过 | 绿色标签"通过" |
| 不通过 | 红色标签"不通过" + 显示不通过原因（背调材料专属） |

### 4.4 技能认证（首页 Tab）

#### 4.4.1 技能认证首页

- 顶部进度概览卡片：展示"已认证 X 项 / 共 Y 项"及环形进度条。
- 技能列表（卡片式）：展示技能名称、认证状态标签（未开始 / 进行中 / 已通过）、科目完成进度（如"2/4 科目已通过"）及进度条。
- 点击技能卡片 → 进入科目列表页。

#### 4.4.2 科目列表（技能详情）

每个科目卡片包含：

| 字段 | 说明 |
|------|------|
| 科目名称 | 如"科目一"～"科目四" |
| 状态标签 | 待考试（灰）/ 考试中（蓝）/ 已通过（绿）/ 未通过（红） |
| 得分 | 已通过时显示分数 |
| 重考次数 | 未通过时显示"剩余重考次数：X 次" |
| 开始考试按钮 | 仅在"待考试"或"未通过且有剩余重考次数"时显示 |

- 底部固定提示文案："科目一与科目四通过后即可获得执照"。
- 执照发放逻辑在执照模块单独设计，本期不展开。

#### 4.4.3 答题页面

| 元素 | 说明 |
|------|------|
| 顶部 | 科目名称 + 倒计时（红色数字，时长由配置决定） |
| 题号指示 | "第 N/总题数 题" |
| 题目区域 | 题目文本 + A/B/C/D 单选卡片，选中态蓝色边框 + 背景 |
| 导航操作 | 上一题 / 下一题 |
| 答题卡 | 弹出底部面板，显示所有题号，已答/未答用不同颜色区分 |
| 交卷 | 点击后弹出二次确认"确认交卷？还有 X 题未作答" |
| 超时自动交卷 | 倒计时结束弹窗"考试时间到，已自动交卷" → 跳转成绩页 |

**考试规则（通过配置控制）：**

- 考试时长限制（分钟）
- 重考次数上限

#### 4.4.4 成绩结果页

| 状态 | 展示 |
|------|------|
| 通过 | 大号绿色分数 + 图标 + "考试通过" + 及格线提示 |
| 不通过 | 大号红色分数 + "很遗憾，未通过" + "剩余重考次数：X 次" |

- 「查看答题详情」按钮 → 进入答题详情页。
- 「返回」按钮 → 返回科目列表。

#### 4.4.5 答题详情页

逐题展示：

- 题目文本
- 所有选项
- 考生所选答案（蓝色标记）
- 正确答案（绿色标记）
- 错误选择（红色标记）
- 底部显示本次总分

### 4.5 考试记录 Tab

## 六、数据规范

### 6.1 执照编号格式

```
省份编码（2位） + 城市编码（2位） + 序列号（5位）
示例：350100001（福建省福州市第1号）
```

### 6.2 考生状态枚举

| 值 | 含义 |
|----|------|
| 有效 | 执照在有效期内 |
| 失效 | 执照失效或未获得执照 |

### 6.3 科目考试状态枚举

| 值 | 含义 |
|----|------|
| 待考试 | 尚未开始考试 |
| 考试中 | 正在进行答题 |
| 已通过 | 考试成绩达到及格线 |
| 未通过 | 考试成绩未达及格线 |

### 6.4 认证材料状态枚举

| 值 | 含义 |
|----|------|
| 待审核 | 考生已提交，管理员尚未处理 |
| 通过 | 管理员审核通过 |
| 不通过 | 管理员审核不通过（附原因） |

### 6.5 考试配置项（可配置）

| 配置项 | 类型 | 说明 |
|--------|------|------|
| 考试时长 | 整数（分钟） | 超时自动交卷 |
| 重考次数上限 | 整数 | 不通过后最多重考次数 |
| 及格分数线 | 整数 | 默认 60 分 |

## 七、非功能性需求

### 7.1 前端

| 要求 | 说明 |
|------|------|
| 考生端 | 移动端 H5，适配主流机型（iOS / Android） |
| 管理端 | PC 宽屏，最小兼容 1280px 宽度 |
| 视觉风格 | 主色调 #1677FF（品牌蓝），辅助色绿/红/黄 |
| 交互反馈 | 按钮 hover/active 态、Toast 提示（顶部滑入，2秒消失）、弹窗动画、页面切换淡入淡出 |

### 7.2 性能

- 页面首屏加载 ≤ 2 秒（4G 网络环境）
- 考试答题页面无明显卡顿，倒计时精度误差 ≤ 1 秒

### 7.3 安全

- 登录态缓存加密存储
- 证件信息（身份证号、手机号）传输加密（HTTPS）
- 接口请求需携带鉴权 Token

### 7.4 可靠性

- 考试进行中页面意外关闭后，重新打开可恢复答题进度
- 提交试卷接口具备幂等性，防止重复提交
