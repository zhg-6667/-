# 执照认证系统 — 交互原型生成提示词

本提示词包含两个部分：考生端（移动端 H5）和后台管理系统（PC Web）。分别生成为独立 HTML 文件，数据通过 localStorage 共享。

---

# 第一部分：H5 移动端考生端

请生成一个完整的「执照认证系统」考生端（移动端 H5）可交互原型，使用单个 HTML 文件实现（内联 CSS + JS）。要求所有页面可真实点击跳转，表单可填写，数据在 localStorage 中持久化模拟，体验接近真实产品。

## 一、全局设计规范

### 视觉风格（参考好慷执照页面）

#### 整体调性
- 设计风格：现代简洁、专业可信，偏向政务/资质认证类应用风格
- 整体干净清爽，大量留白，信息层级清晰

#### 配色方案
- **主色调**：#3B82F6（品牌蓝，用于顶部头卡背景、主按钮、强调数字）
- **头部渐变**：linear-gradient(135deg, #4A9CF6 0%, #3B82F6 50%, #2563EB 100%)，顶部个人信息卡片使用蓝色渐变背景 + 白色文字
- **辅助色**：#52C41A（成功绿，用于"生效"标签、认证勾选图标）、#FF4D4F（错误红）、#FAAD14（警告黄）、#FF8C00（未通过橙色）
- **页面背景**：#F5F7FA（浅灰），内容区域与蓝色头部形成对比
- **卡片背景**：#FFFFFF
- **文字颜色**：#333（主文字）、#666（次要文字）、#999（辅助说明文字）

#### 顶部头卡区域（蓝色区域）
- 蓝色渐变圆角大卡片，圆角约 16px，内含：
  - 品牌标识 "Homeking"（白色，居中顶部）
  - 左侧：圆形头像（68px，2px 白色描边）
  - 右侧：姓名（白色，22px 粗体）+ 角色标签（白色半透明底，如"服务人员"）+ 手机号
  - 右上角：半透明水印/印章装饰图（圆形徽章纹样，opacity 0.15）
- 头卡下方紧接认证图标栏：三个认证项横向等分排列
  - 每项：蓝色图标（带圆形蓝色底）+ 右下角绿色小勾 + 底部文字说明
  - 图标样式：线性图标 + 圆角矩形蓝色背景

#### 卡片样式
- 圆角：12px（大卡片）、8px（小卡片/标签）
- 阴影：0 2px 12px rgba(0,0,0,0.06)，非常柔和
- 内间距：16-20px
- 卡片间距：12px

#### 字体规范
- 系统默认字体栈：-apple-system, "PingFang SC", "Microsoft YaHei", sans-serif
- 姓名/大标题：22px, font-weight: 700
- 分数数字：32px, font-weight: 700, 品牌蓝色
- 分数单位/总分：14px, #999
- 模块标题（如"专业技能"）：18px, font-weight: 600, 左侧带 4px 蓝色竖条装饰
- 正文/字段标签：14-15px, #666
- 字段值：14-15px, #333, 右对齐

#### 标签/徽章样式
- "生效"标签：绿色背景（#E8F5E9）+ 绿色文字 + 左侧绿色小圆点/勾，圆角 12px（胶囊形）
- 分数展示：大号蓝色数字 + 小号灰色"/总分" + 右箭头，可点击

#### 信息展示方式
- 键值对排列：左侧标签灰色，右侧值黑色，两端对齐（justify-content: space-between）
- 分割线：1px solid #F0F0F0，非常淡
- 列表项带右箭头"›"，暗示可点击

#### 技能卡片样式
- 白色卡片，圆角 12px
- 左侧：蓝色背景圆角图标（技能类别图标）
- 标题：16px 粗体
- 底部：胶囊形标签显示进度（浅蓝底 #EBF5FF + 蓝色文字，其中数字加粗变色）
- 右侧可带箭头表示可展开/跳转

#### 装饰元素
- 顶部蓝色卡片右上角有半透明圆形印章/徽章纹样（作为背景装饰）
- 整体风格避免过多装饰，以信息展示为主

### 交互规范
- 所有按钮有 hover/active 态反馈
- 表单输入有 focus 高亮边框
- 操作成功/失败有 Toast 提示（顶部滑入，2秒自动消失）
- 弹窗有半透明遮罩 + 居中卡片 + 进入动画
- 页面切换有淡入过渡动画（transition 300ms）
- 列表项有点击涟漪效果

### 布局框架
- 页面居中灰色背景，手机模拟框 375×750px（黑色圆角边框 + 顶部刘海）
- 底部 Tab 导航（4个）：练习题库 | 技能认证 | 我的执照 | 个人中心
- 默认进入"练习题库"Tab
- JS 页面栈管理 show/hide 切换，支持返回

### 页面 1：练习首页（底部 Tab - 练习题库，默认页）

- **科目考试信息卡片**：
  - 展示：题目总数（如 100 题）、模拟考试及格线（如 90 分）、正式考试及格线（如 90 分）及次数限制（如仅 2 次机会）
  - "开始模拟考试"按钮 → 进入模拟考试（页面 5）

  - 点击 → 进入题库练习页（页面 3），自动筛选该场景的题目

### 页面 2：题库练习页

- 导航栏：返回箭头 + 标题"题库练习"
- **统计概览栏**（三个数字卡片横排）：
  - 总题数（蓝色大字）| 已答题（绿色大字）| 未做题（灰色大字）
- **"全部练习"按钮**（主按钮）→ 进入答题页（页面 3），从头开始
- **按知识点练习**（模块标题带蓝色竖条）：
  - 知识点分类列表（Cell 列表样式，带右箭头）：

### 页面 3：练习答题页（即时反馈，无计时）

- 顶部栏：返回按钮 + 标题"题库练习" + 右侧"答题卡"按钮
- **题目区域**：
  - 题目类型标签（胶囊形）：单选 / 多选 / 图片题 / 判断题
  - 题目文本（16px，行高 1.6）
  - 图片题时：题目下方展示图片（灰色占位框模拟）
  - 选项列表：
    - 单选题：圆形单选按钮 + 选项文本
    - 多选题：方形勾选框 + 选项文本
    - 判断题：对/错 两个选项
  - 选中态：蓝色边框 + 浅蓝背景
- **即时反馈**（选择答案后立即显示）：
  - 回答正确 → 选项变绿色 + 绿色提示条"回答正确 ✓"，1 秒后自动进入下一题
  - 回答错误 → 错误选项变红 + 正确选项变绿 + 红色反馈区域：
    - ✗ 回答错误
    - 正确答案：B. xxx
    - 你的答案：A. xxx
    - 知识点解析文本
    - "下一题"按钮手动跳转
- **底部状态栏**（固定在底部）：
  - 左侧：绿色圆点 + 正确数 | 红色圆点 + 错误数
  - 右侧：当前题号 / 总题数（如 1/100）
- **答题卡面板**（点击顶部"答题卡"按钮弹出，底部滑入面板）：
  - 题号网格（每行 5~8 个）
  - 颜色区分：绿色 = 正确、红色 = 错误、灰色 = 未答、蓝色边框 = 当前题
  - 点击题号可跳转
- **完成所有题目** → 自动进入练习结果页（页面 4A）

### 页面 4：模拟考试答题页（有倒计时，不显示对错）

- 底部操作栏：「上一题」「答题卡」「下一题/交卷」
- 答题卡面板：已答（蓝色）/ 未答（灰色）
- 交卷逻辑：

  - 倒计时结束 → 弹窗"考试时间到，已自动交卷"
- 交卷后 → 进入模拟考试结果页（类似页面 4A 但标题为"模拟考试结果"）

### 页面 5：登录页

- 顶部：App 图标（蓝色盾牌+对勾 SVG）+ 文案"执照认证平台" + 副标题"职业技能认证一站式服务"
- **手机号输入框**（带手机图标前缀）
- **图形验证码区域**：左侧输入框 + 右侧验证码图片（用 canvas 随机生成 4 位数字，点击可刷新）
- **短信验证码区域**：左侧输入框 + 右侧"获取验证码"按钮
  - 图形验证码正确后按钮才可点击

- **登录/注册按钮**（蓝色渐变主按钮）：
  - 以手机号为唯一标识，未注册则自动注册
  - 已有账号 → 验证码校验（模拟：任意 4 位即通过）→ 登录成功 Toast → 跳回触发页
  - 新账号 → 注册成功 → 进入信息完善引导流程（页面 6）
- **底部协议勾选**：「我已阅读并同意《用户服务协议》」
  - 协议文字可点击弹窗查看完整内容
  - 未勾选时点击登录 → 弹窗提示"请确认勾选协议"，弹窗有「确认」按钮，点击后自动勾选

### 页面 6：信息完善 / 材料补全引导页

复用同一页面，支持两种模式：
- **新用户模式**（注册后）：展示全部 5 个步骤，标题"完善信息"
- **补全模式**（材料检查未通过时）：仅展示缺失材料的步骤，标题"补全材料"，完成后自动返回技能报名流程

分步引导流程，顶部有步骤进度条，每步可"下一步"或"跳过"：

- **步骤 1：账号信息录入**
  - 头像上传（点击圆形区域模拟上传）
  - 昵称输入框
  - 个人照片上传 → 上传后模拟 AI 人像验证：
    - 验证中：加载动画 + "AI人像验证中..."
    - 验证成功：绿色对勾 + "验证通过"
    - 验证失败：红色提示（随机模拟："未检测到人脸"/"检测到多人"/"照片模糊"）+ "重新上传"按钮
- **步骤 2：身份证上传**
  - 正反面两个上传框（人像面 + 国徽面）
  - 上传后模拟 OCR 识别，自动填充：姓名、证件号、有效期
  - 有效期过期 → 红色标签"已过期"
- **步骤 3：健康证上传**
  - 单张上传框
  - 上传后模拟 OCR 识别有效期
  - 上传后 AI 自动审核（模拟 1.5 秒延迟，90% 通过率）：
    - 通过 → 绿色提示"AI审核通过"
    - 不通过 → 红色提示"审核不通过，请重新上传"
- **步骤 4：体检报告上传**
  - 上传框
  - 上传后 AI 自动审核（模拟 1.5 秒延迟，90% 通过率）：
    - 通过 → 绿色提示"AI审核通过"
    - 不通过 → 红色提示"审核不通过，请重新上传"
- **步骤 5：背调验证（第三方接口）**
  - 输入姓名 + 身份证号（可从步骤2自动带入）
  - "开始验证"按钮 → 模拟调用第三方接口：
    - 验证中：加载动画 + "背调查询中..."
    - 验证通过：绿色提示"背调验证通过"，**接口确认即审核通过**（无需人工复核）
    - 验证不通过：红色提示 + 不通过原因 + "重新验证"按钮
- **完善完成** → Toast"资料提交成功" → 跳回触发登录的原页面

### 页面 7：技能选择页（底部 Tab - 技能认证，需登录）

- 点击Tab时检查登录态 → 未登录跳登录页（页面 5）→ 已登录显示本页
- 导航栏：标题"技能报考认证"（Tab页无返回箭头）
- **技能列表**（卡片式），每个技能卡片：
  - 左侧：彩色圆角图标（技能类别）
  - 技能名称（16px 粗体）
  - 支付状态标签：
    - 未支付 → 灰色标签"未报名"
    - 已支付 → 蓝色标签"已报名"
    - 全科通过 → 绿色标签"已通过"
  - 科目进度（如"2/4 科目已通过"）+ 进度条
  - 右侧箭头
  - 点击逻辑：
    - 先检查材料是否齐全（个人照片+身份证+健康证+体检报告已上传，背调验证已完成）
      - 未齐全 → 直接进入"补全材料"引导页（复用页面 6 的分步流程，仅展示缺失项的步骤，在单页内依次完成，不跳转到各个模块）→ 补全完成后自动返回继续报名支付
    - 材料齐全 + 未支付 → 进入支付页（页面 8）
    - 已支付 → 进入科目列表（页面 9）

### 页面 8：支付认证费页

- 导航栏：返回箭头 + 标题"确认支付"
- **订单信息卡片**：
  - 技能名称
  - 认证费用金额（大号蓝色数字，如 ¥199）
  - 包含内容说明（如"含科目一~四考试各 1 次机会，重考另收费"）
- **支付方式**：微信支付（模拟，单选选中态）
- **底部按钮**："确认支付 ¥199"（蓝色主按钮）
  - 点击 → 模拟支付中加载 → 随机成功/失败（90% 成功）
  - 支付成功 → Toast"支付成功" → 跳转科目列表（页面 9）
  - 支付失败 → 弹窗"支付失败，请重试" → 可重试或返回

### 页面 9：科目列表（技能详情）

- 导航栏：返回箭头 + 技能名称
- **材料审核状态卡片**（顶部，根据审核状态展示不同样式）：
  - 全部通过 → 绿色卡片"材料审核已通过，可以开始考试"
  - 审核中 → 黄色卡片"材料审核中，请等待审核通过后参加考试"，列出各项审核状态
  - 有不通过 → 红色卡片"部分材料审核不通过"+ 不通过项 + "去修改 ›"跳转链接
  - 审核项目：个人照片（人工）、身份证（人工）、健康证（AI自动）、体检报告（AI自动）、背调验证（接口自动）
- **科目卡片列表**，每个科目：
  - 科目名称（如"科目一：理论基础"）
  - 状态标签（胶囊形）：待考试（灰）/ 已通过（绿）/ 未通过（红）
  - 已通过 → 显示得分（蓝色大字）
  - 未通过 → 显示得分（红色）+ "剩余重考次数：X 次"
  - 未通过且重考次数为 0 → 显示"支付重考费"按钮（橙色）

## 四、技术要求

1. **单文件实现**：所有 HTML + CSS + JS 在一个文件中，`<meta charset="UTF-8">`
2. **路由模拟**：用 JS 控制 div 的 show/hide 实现页面切换，维护页面栈支持返回
3. **数据持久化**：所有数据操作（增删改查）通过 localStorage 模拟
4. **手机模拟框**：375×750px，黑色圆角边框 + 顶部刘海，居中灰色背景
5. **底部 Tab**：练习题库 | 技能认证 | 我的执照 | 个人中心，Tab 切换不入栈，默认显示练习题库
6. **动画**：页面切换淡入、弹窗缩放进入、Toast 滑入滑出、按钮点击反馈
7. **无外部依赖**：不引入任何 CDN 或外部资源，图标用 SVG 内联
8. **canvas 验证码**：随机 4 位数字 + 干扰线 + 噪点
9. **考试倒计时**：实时计时，超时自动交卷
10. **练习即时反馈**：答题后立即显示对错 + 知识点解析
11. **支付模拟**：加载动画 + 随机成功/失败结果

1. **单文件实现**：所有 HTML + CSS + JS 在一个 `prototype-admin.html` 文件中
2. **全屏布局**：左侧 220px 固定侧边栏 + 右侧自适应内容区，无手机模拟框
3. **菜单切换**：点击侧边栏菜单项切换右侧内容区页面，选中态高亮
4. **数据持久化**：所有数据通过 localStorage 模拟，与考生端 H5 共享同一套 key
5. **表格分页**：每页 5-10 条，支持翻页
6. **弹窗系统**：通用弹窗组件，支持自定义标题/内容/按钮
7. **搜索筛选**：实时过滤表格数据
8. **无外部依赖**：不引入任何 CDN 或外部资源，图标用 SVG 内联
9. **响应式**：最小宽度 1200px，内容区自适应
10. **操作反馈**：增删改操作后 Toast 提示 + 表格刷新
11. **数据联动**：后台修改执照状态/审核状态等，考生端数据同步变更（通过共享 localStorage）
