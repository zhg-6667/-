# 执照认证系统 — 全交互原型生成提示词

请生成一个完整的「执照认证系统」可交互原型，使用单个 HTML 文件实现（内联 CSS + JS），包含考生端（移动端H5）和管理端（PC后台）双端完整流程。要求所有页面可真实点击跳转，表单可填写，数据在 localStorage 中持久化模拟，体验接近真实产品。

---

## 一、全局设计规范

### 视觉风格（参考好慷执照页面）

#### 整体调性
- 设计风格：现代简洁、专业可信，偏向政务/资质认证类应用风格
- 整体干净清爽，大量留白，信息层级清晰

#### 配色方案
- **主色调**：#3B82F6（品牌蓝，用于顶部头卡背景、主按钮、强调数字）
- **头部渐变**：linear-gradient(135deg, #4A9CF6 0%, #3B82F6 50%, #2563EB 100%)，顶部个人信息卡片使用蓝色渐变背景 + 白色文字
- **辅助色**：#52C41A（成功绿，用于"生效"标签、认证勾选图标）、#FF4D4F（错误红）、#FAAD14（警告黄）
- **页面背景**：#F5F7FA（浅灰），内容区域与蓝色头部形成对比
- **卡片背景**：#FFFFFF
- **文字颜色**：#333（主文字）、#666（次要文字）、#999（辅助说明文字）

#### 顶部头卡区域（蓝色区域）
- 蓝色渐变圆角大卡片，圆角约 16px，内含：
  - 品牌标识 "Homeking"（白色，居中顶部）
  - 左侧：圆形头像（68px，2px 白色描边）
  - 右侧：姓名（白色，22px 粗体）+ 角色标签（白色半透明底，如"保洁师"）+ 工号
  - 右上角：半透明水印/印章装饰图（圆形徽章纹样，opacity 0.15）
- 头卡下方紧接认证图标栏：三个认证项横向等分排列
  - 每项：蓝色图标（带圆形蓝色底）+ 右下角绿色小勾 + 底部文字说明
  - 图标样式：线性图标 + 圆角矩形蓝色背景

#### 卡片样式
- 圆角：12px（大卡片）、8px（小卡片/标签）
- 阴影：0 2px 12px rgba(0,0,0,0.06)，非常柔和
- 内间距：16-20px
- 卡片间距：12px

#### 字体规范
- 系统默认字体栈：-apple-system, "PingFang SC", "Microsoft YaHei", sans-serif
- 姓名/大标题：22px, font-weight: 700
- 分数数字：32px, font-weight: 700, 品牌蓝色
- 分数单位/总分：14px, #999
- 模块标题（如"专业技能"）：18px, font-weight: 600, 左侧带 4px 蓝色竖条装饰
- 正文/字段标签：14-15px, #666
- 字段值：14-15px, #333, 右对齐

#### 标签/徽章样式
- "生效"标签：绿色背景（#E8F5E9）+ 绿色文字 + 左侧绿色小圆点/勾，圆角 12px（胶囊形）
- 分数展示：大号蓝色数字 + 小号灰色"/总分" + 右箭头，可点击

#### 信息展示方式
- 键值对排列：左侧标签灰色，右侧值黑色，两端对齐（justify-content: space-between）
- 分割线：1px solid #F0F0F0，非常淡
- 列表项带右箭头"›"，暗示可点击

#### 技能卡片样式
- 白色卡片，圆角 12px
- 左侧：蓝色背景圆角图标（技能类别图标）
- 标题：16px 粗体
- 底部：胶囊形标签显示服务时长（浅蓝底 #EBF5FF + 蓝色文字，其中数字加粗变色）
- 右侧可带箭头表示可展开/跳转
- 卡片右侧可有装饰性插图（如"员工档案"浮动气泡）

#### 装饰元素
- 顶部蓝色卡片右上角有半透明圆形印章/徽章纹样（作为背景装饰）
- 右下角可有浮动功能入口气泡（如"员工档案"，带小图标 + 文字，圆角，带阴影）
- 整体风格避免过多装饰，以信息展示为主

### 交互规范
- 所有按钮有 hover/active 态反馈
- 表单输入有 focus 高亮边框
- 操作成功/失败有 Toast 提示（顶部滑入，2秒自动消失）
- 弹窗有半透明遮罩 + 居中卡片 + 进入动画
- 页面切换有淡入过渡动画（transition 300ms）
- 列表项有点击涟漪效果

### 双端入口
- 页面顶部设置端切换 Tab：「考生端（H5）」|「管理端（PC）」
- 考生端用手机模拟框包裹（375×667），居中展示
- 管理端全宽展示，左侧导航 + 右侧内容区

## 二、考生端（移动端 H5）

### 页面 1：登录页
- 顶部：App 图标（蓝色盾牌+对勾 SVG）+ 文案"执照认证平台"
- 手机号输入框（带手机图标前缀）
- 图形验证码区域：左侧输入框 + 右侧验证码图片（用 canvas 随机生成4位数字，点击可刷新）
- 短信验证码区域：左侧输入框 + 右侧"获取验证码"按钮
  - 图形验证码正确后按钮才可点击
  - 点击后60秒倒计时
- 底部协议勾选：「我已阅读并同意《用户服务协议》」，协议文字可点击弹窗查看
- 登录/注册按钮：
  - 未勾选协议时点击 → 弹窗提示"请确认勾选协议"，弹窗有「确认」按钮，点击后自动勾选
  - 以手机号为唯一标识，未注册则自动注册
  - 验证码模拟校验（任意4位即通过）→ 登录成功 Toast → 跳转个人中心

### 页面 2：个人中心（Tab首页）
- 底部 Tab 导航栏（3个tab）：首页（技能认证）| 考试记录 | 我的
- **「我的」Tab 内容：**
  - 顶部用户卡片：头像（圆形）、昵称、手机号
  - 功能列表（带右箭头的 Cell 列表）：
    - 账号信息
    - 证件信息
    - 认证材料
    - 退出登录

### 页面 3：账号信息
- 导航栏：返回箭头 + 标题"账号信息"
- 表单项（点击可编辑）：
  - 头像：圆形图片，点击可模拟上传（用 FileReader 预览）
  - 昵称：可编辑文本
  - 手机号：显示（带掩码 138****1234）
  - 个人照片：上传区域（虚线框+相机图标），上传后模拟 AI 人像验证
    - 验证中：加载动画 + "AI人像验证中..."
    - 验证成功：绿色对勾 + "验证通过"
    - 验证失败：红色提示具体原因（随机模拟："未检测到人脸"/"检测到多人"/"照片模糊"），显示"重新上传"按钮
- 底部「保存」按钮

### 页面 4：证件信息
- 导航栏：返回箭头 + 标题"证件信息"
- 两个证件卡片，每个卡片包含：
  - **身份证**：上传区域（正反面两个框），上传后模拟 OCR 识别，自动填充：
    - 姓名、证件号（与账号信息交叉校验，不一致弹窗警告"证件信息与账号不符"）
    - 有效期（自动识别），过期则红色标签"已过期"
  - **健康证**：上传区域，上传后自动识别有效期
    - 有效期内：绿色标签"有效"
    - 已过期：红色标签"已过期，请重新上传"
- 任何时候都可以重新上传覆盖

### 页面 5：认证材料
- 导航栏：返回箭头 + 标题"认证材料"
- 两个认证卡片：
  - **体检报告**：
    - 未上传：上传按钮
    - 已上传待审核：黄色标签"待审核"，预览缩略图
    - 审核通过：绿色标签"通过"
    - 审核不通过：红色标签"不通过"
  - **背调报告**：
    - 同上逻辑
    - 不通过时额外显示原因文本

### 页面 6：技能认证首页（首页Tab）
- 导航栏：标题"技能认证"
- 顶部进度概览卡片：
  - "已认证 X 项技能 / 共 Y 项"
  - 环形进度条
- 技能列表（卡片式）：
  - 每个技能卡片包含：技能名称（如"保洁"、"收纳"、"烹饪"）、认证状态标签（未开始/进行中/已通过）、科目完成进度（如"2/4 科目已通过"）、进度条
  - 点击卡片 → 进入该技能的科目列表

### 页面 7：科目列表（技能详情）
- 导航栏：返回箭头 + 技能名称
- 科目卡片列表，每个科目：
  - 科目名称（如"科目一"、"科目二"、"科目三"、"科目四"）
  - 状态标签：待考试（灰色）/ 考试中（蓝色）/ 已通过（绿色）/ 未通过（红色）
  - 已通过的显示得分
  - 未通过的显示"剩余重考次数：X 次"
  - 「开始考试」按钮（仅待考试/未通过且有重考次数时显示）
- 底部提示："科目一与科目四通过后即可获得执照"

### 页面 8：答题页面
- 顶部：科目名称 + 倒计时（红色数字，模拟 30:00 倒计时）
- 题目区域：
  - 题号指示："第 1/10 题"
  - 题目文本
  - 选项列表（A/B/C/D 单选卡片），选中态有蓝色边框+背景
- 底部操作栏：
  - 「上一题」「下一题」按钮
  - 答题卡按钮（点击弹出底部答题卡面板，显示所有题号，已答/未答用不同颜色区分）
  - 「交卷」按钮 → 二次确认弹窗"确认交卷？还有X题未作答"
- 倒计时结束自动弹窗提示"考试时间到，已自动交卷" → 跳转成绩页

### 页面 9：成绩结果页
- 大号分数展示（居中，绿色通过/红色不通过）
- 及格线提示（如"及格线：60分"）
- 通过：🎉 恭喜图标 + "考试通过"
- 不通过："很遗憾，未通过" + "剩余重考次数：X 次"
- 「查看答题详情」按钮 → 进入详情页
- 「返回」按钮

### 页面 10：答题详情
- 逐题展示：题目 + 所有选项 + 考生选择（蓝色标记）+ 正确答案（绿色标记）+ 错误选择（红色标记）
- 底部显示总分

### 页面 11：考试记录（考试记录Tab）
- 按时间倒序展示所有考试记录
- 每条记录卡片：技能名称、科目名称、考试时间、得分、状态标签
- 点击 → 进入该次考试的答题详情

## 五、技术要求

1. **单文件实现**：所有 HTML + CSS + JS 在一个文件中
2. **路由模拟**：用 JS 控制 div 的 show/hide 实现页面切换，维护页面栈支持返回
3. **数据持久化**：所有数据操作（增删改查）通过 localStorage 模拟
4. **双端联动**：管理端审核通过后，考生端认证状态实时更新（同一份 localStorage 数据）
5. **响应式**：考生端固定手机模拟框，管理端适配宽屏
6. **动画**：页面切换淡入淡出、弹窗缩放进入、Toast 滑入滑出、按钮点击反馈
7. **无外部依赖**：不引入任何 CDN 或外部资源，图标用 SVG 内联或 CSS 绘制
