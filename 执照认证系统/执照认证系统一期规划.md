# 执照认证系统 - 一期 MVP 功能模块规划

## Context

基于《26年3月-执照认证系统》完整设计文档，规划最小 MVP 版本。目标是跑通核心链路：**用户登录 → 刷题练习 → 报考 → 线上考试 → 查看成绩**，同时后台支持**全流程代报名**。

### 关键裁剪决策
- **材料审核**：极简版，只做基本信息+身份证上传（接入腾讯云OCR自动提取），人工审核通过即可
- **支付**：暂不接入微信支付，后台直接标记已支付状态
- **考试科目**：只做线上笔试类科目，实操类科目后续再加
- **代报名**：后台全流程代办（添加考生、上传材料、报名、标记审核）

## 一期功能模块总览

### 移动端（H5）

#### 模块1：用户登录/注册

**1.1 登录页**
- 手机号输入
- 图形验证码（点击可刷新，验证通过后"获取验证码"按钮可点击）
- 短信验证码输入
- 新手机号自动注册，无需单独注册流程
- 登录成功后返回触发登录的页面

**1.2 新用户信息完善页（首次登录触发）**
- 填写姓名、昵称
- 完善后进入首页

**业务规则：**
- 手机号为唯一标识
- 登录状态缓存（token），过期需重新登录
- 未登录访问受限页面 → 跳转登录页 → 登录后回到原页面

**砍掉**：微信授权登录、协议弹窗

#### 模块2：练习题库（需登录）

> MVP 改为需登录才能刷题，做题进度全部存服务端。所有启用课程都可练习，无需报考。

**2.1 练习首页**
- 顶部课程选择器：切换后展示该课程关联的考试科目和题库
- 进度卡片：做题进度环形图、已做题数、累计刷题数、正确率
- "开始做题"按钮 → 进入题库练习页
- 快捷入口：随机练习（随机抽固定数量题，如20题） | 错题回顾 | 模拟考试
- 科目考试信息卡片：题目总数、及格线信息展示

**2.2 题库练习页**
- 统计概览：总题数 | 已答题 | 未做题
- "全部练习"按钮（顺序做所有题）
- 针对性刷题：选定课程后可按"考试科目 + 分类"两层筛选
- 题目范围 = 考试绑定的题目（手动选题=固定题目，随机抽题=该分类下全部题目）

**2.3 答题页**
- 题目类型：单选（单选按钮）| 多选（勾选框）| 判断题
- 题目支持媒体附件（图片/视频/GIF），媒体渲染在题干文字下方
- 答题后即时反馈：
  - 正确 → 绿色提示，自动进入下一题
  - 错误 → 红色提示 + 正确答案 + 你的答案 + 知识点解析
- 底部状态栏：正确数 · 错误数 · 当前题号/总题数
- 答题卡面板：题号网格，已答/未答/正确/错误颜色区分

**2.4 练习结果页**
- 统计：总答题数 | 错误数 | 正确率
- 操作按钮：回顾错题 | 继续答题

**2.5 模拟考试**
- 流程与正式考试一致（倒计时、答题卡、交卷），共用 exam_record 表加 is_mock 字段
- 不消耗重考次数
- 不需要人脸验证
- 不启用切屏检测

**砍掉**：知识点场景学习

#### 模块3：课程认证（需登录）

**3.1 认证中心页**
- 展示所有启用课程的认证进度卡片，按三种状态分组展示：
  - **未认证**（灰色标签）：未报考，显示课程名称、描述、科目数，点击进入材料提交页
  - **认证中**（蓝色标签）：已报考但未全部通过，显示当前阶段细节：
    - 材料审核中（audit_status=0）→ 显示"材料审核中，请耐心等待"
    - 材料审核不通过（audit_status=2）→ 显示"审核不通过"+ 原因，点击重新提交材料
    - 考试进行中（audit_status=1 且 exam_status≠2）→ 卡片内展开各科目列表：
      - 已通过：绿色标签 + 分数
      - 未通过（有重考次数）：橙色标签 + "去考试"按钮，点击直接进入考试流程
      - 未通过（无重考次数）：灰色标签 + "次数已用完"
      - 未考：灰色标签 + "去考试"按钮
  - **已认证**（绿色标签）：所有科目通过（exam_status=2），显示认证通过时间

**3.2 材料提交页**
- 身份证正面上传 → 腾讯云OCR自动提取姓名、身份证号、性别、出生日期 → 填充表单 → 考生确认后提交
- 基本信息确认（姓名、手机号、身份证号、性别、出生日期，OCR提取+账号信息带入）
- 个人照片上传
- 提交后显示"待审核"状态

**3.3 审核状态页**
- 待审核 → 提示等待
- 审核通过 → 可进入考试科目列表
- 审核不通过 → 显示不通过原因，可重新上传材料

**业务规则：**
- 材料审核通过后直接可考试，无单独报名审核环节
- 不接入支付，后台如需标记付费状态可在考生管理中操作

**砍掉**：健康证、体检报告、背调验证、AI审核、支付、报名审核

#### 模块4：线上考试（需登录 + 材料审核通过）

**4.1 科目列表页**
- 前置条件：已报考课程 + 材料审核通过
- 展示当前课程下的笔试科目列表
- 每个科目显示：名称、状态（未考/已通过/未通过）、已通过显示分数
- 每个科目参数独立配置（时长、及格分、重考次数）

**4.2 考前人脸识别**
- 点击"开始考试" → 调用腾讯云人脸核身SDK进行人脸识别
- 通过后进入答题页
- 失败可重试3次，超过则禁止本次考试
- 模拟考试不需要人脸验证

**4.3 考试答题页**
- 题目类型：单选/多选/判断
- 题目支持媒体附件（图片/视频/GIF），媒体渲染在题干文字下方
- 顶部倒计时（科目配置的考试时长），超时自动交卷
- 答题过程中不显示对错，仅记录答案
- 答题卡面板：已答/未答颜色区分
- 交卷确认弹窗："确定交卷？未答题目将不计分"
- 防作弊：H5监听visibilitychange，每次切回弹窗警告，超过阈值（后台可配置，默认2次）自动交卷

**4.4 考试结果页**
- 通过：考试名称 + 绿色大号分数 + 总题数 + 错误数 + "恭喜你完成了本次考试！"
- 未通过：橙色大号分数 + 总题数 + 错误数 + "考试不通过，建议回顾知识后再来考试"
  - 剩余重考次数 > 0 → "重新考试"按钮
  - 剩余重考次数 = 0 → 提示联系管理员

**业务规则：**
- 中途退出= 直接交卷，按已答题目结算
- 重考次数由后台按科目配置，次数用完需管理员重置
- 不接入重考付费

**砍掉**：重考付费、证书查看、执照申领、断点续答

#### 模块4-ext：考试申诉

**4-ext.1 申诉入口**
- H5考试记录页，所有"未通过"的考试记录增加"申诉"按钮
- 不限申诉次数

**4-ext.2 申诉流程**
- 考生填写申诉理由提交
- 后台审核：通过 → 返还1次重考机会；驳回 → 显示驳回理由

#### 模块5：考试记录（需登录）

**5.1 成绩列表页**
- 历史考试成绩列表（时间倒序）
- 列表字段：科目名称、得分、及格分、结果（通过/未通过）、考试时间
- 点击进入答题详情

**5.2 答题详情页**
- 逐题展示：题目、考生答案、正确答案、是否正确、知识点解析

#### 模块6：个人中心（需登录）

**6.1 个人中心主页**
- 头部信息卡：头像、姓名、手机号
- 功能列表：
  - 账号信息（查看/编辑头像、昵称、姓名）
  - 认证材料（身份证、个人照片的审核状态，可重新上传）
  - 考试记录（入口，跳转模块5）
  - 退出登录

**砍掉**：修改手机号

### 管理后台

#### 模块A：题目库管理

**A1. 分类管理页**
- 分类列表：名称、题目数量、排序、状态
- 新建/编辑/删除分类
- 分类排序（影响前端题库练习页的展示顺序）

**A2. 题目列表页**
- 列表字段：题目编号（自动生成）、题目内容（截断）、类型（单选/多选/判断）、分类、创建人、创建时间、状态（生效/失效）
- 搜索：题目内容关键词
- 筛选：分类、状态、时间段
- 操作：编辑、删除、启用/禁用
- 不设难度字段

**A3. 新建/编辑题目页**
- 选择题目类型：单选 / 多选 / 判断
- 题目内容（富文本编辑器）
- 媒体附件上传（可选）：支持图片（jpg/png，≤5MB）、视频（mp4，≤10MB，≤60秒）、GIF（≤2MB），渲染在题干文字下方
- 设置选项（MVP 纯文本选项，图片选项后续）
- 标记正确答案（单选选一个，多选选多个，判断选对/错）
- 选择分类（从分类管理中动态获取）
- 设置分值
- 知识点解析（答错时展示给学员）

**A4. 批量导入/导出**
- Excel 模板下载
- Excel 批量导入（校验格式、类型、正确答案，导入结果反馈；批量导入暂不支持媒体附件）
- 题目导出 Excel

#### 模块B：考试管理（含成绩中心，合并）

**B1. 考试配置（Tab1）**
- 考试列表：名称、关联课程/科目、题目数、总分、及格分、时长、重考次数、切屏次数上限、状态
- 新建/编辑考试：
  - 基础信息：名称、描述、总分、及格分、考试时长、重考次数限制、切屏次数上限（默认2）
  - 组卷方式（二选一）：
    - 手动选题：从题库按分类筛选并勾选题目
    - 随机抽题：按分类设置抽题比例（%），向下取整，差额从未配置分类随机补；题目不足时页面警告不拦截，有多少抽多少差额从其他分类补
- 启用/停用考试

**B2. 考试记录（Tab2，原成绩中心合并至此）**
- 列表字段：考试编号、考生姓名、手机号、课程、科目、得分、及格分、结果（通过/未通过）、考试时间
- 搜索：考生姓名、手机号
- 筛选：课程、科目、考试结果、时间段
- 操作：查看答题详情（弹窗逐题展示）
- 导出成绩 Excel

**B3. 申诉管理（Tab3）**
- 列表字段：考生姓名、手机号、考试名称、得分、申诉理由、状态（待审核/已通过/已驳回）、提交时间
- 操作：通过（返还1次重考机会）/ 驳回（填写驳回理由）

#### 模块C：考生管理（含代报名）

**C1. 考生列表页**
- 列表字段：账号ID、头像、昵称、姓名、手机号、状态（有效/失效）
- 搜索：姓名/手机号
- 筛选：状态
- 操作：编辑、查看考试详情

**C2. 添加考生（代报名核心）**
- 录入基本信息：姓名、手机号、昵称（系统自动以手机号创建账号，考生可通过手机验证码登录）
- 上传材料：身份证正反面、个人照片
- 直接标记材料审核通过（跳过审核流程）
- 选择课程并为考生报名
- 不发短信验证码，改弹窗二次确认（"确认添加该考生？"）

**C3. 编辑考生**
- 修改基本信息、切换状态

**C4. 考生考试详情**
- 查看该考生的所有考试记录和成绩

#### 模块C-ext：课程管理

**课程列表页**
- 列表字段：课程名称、关联科目数、状态（启用/停用）
- 新建/编辑课程：名称、描述、关联考试科目（多对多，同一科目可被多个课程共用）
- 启用/停用

#### 模块D：认证审核

**D1. 审核列表页**
- 筛选标签：全部 / 待审核 / 已通过 / 不通过
- 列表字段：考生姓名、手机号、材料类型（身份证/个人照片）、提交时间、状态、材料预览
- 操作：通过 / 不通过（填写不通过原因）

**业务规则：**
- 材料审核通过即可考试，无单独报名审核环节
- 代报名添加的考生可直接标记审核通过

### 一期不做的模块（二期规划）

| 模块 | 原因 |
|------|------|
| 执照申领/管理 | 需要全部科目通过后的后续流程 |
| 违规管理/执照分 | 依赖执照模块 |
| 数据看板 | 非核心，有数据后再做 |
| 微信支付 | MVP 先由后台标记支付状态 |
| AI审核/背调 | 第三方接口集成，MVP 用人工审核替代 |
| 场景学习 | 内容运营功能，非必须 |
| 微信授权登录 | 先用手机验证码，后续加微信 |
| 实操类科目 | 需线下配合，后续再加 |
| 角色权限控制 | MVP 先用单一管理员角色 |

## 验证方式

MVP 上线后验证的核心链路：
1. **学员自助链路**：注册登录 → 刷题练习 → 提交报考材料 → 审核通过 → 参加笔试 → 查看成绩
2. **代报名链路**：后台添加考生 → 上传材料 → 标记审核通过 → 为考生报名 → 考生登录参加考试
3. **管理链路**：录入题目 → 配置考试 → 审核材料/报名 → 查看成绩导出

### 数据模型设计

#### 1. 用户与认证

```sql
-- 用户表
CREATE TABLE user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    phone VARCHAR(11) NOT NULL UNIQUE COMMENT '手机号，唯一标识',
    nickname VARCHAR(50) COMMENT '昵称',
    real_name VARCHAR(50) COMMENT '真实姓名',
    avatar_url VARCHAR(500) COMMENT '头像URL',
    id_card_no VARCHAR(18) COMMENT '身份证号',
    gender TINYINT COMMENT '性别 1=男 2=女',
    birth_date DATE COMMENT '出生日期',
    status TINYINT DEFAULT 1 COMMENT '1=有效 0=失效',
    is_info_completed TINYINT DEFAULT 0 COMMENT '是否已完善信息',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 认证材料表
CREATE TABLE user_material (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    material_type TINYINT NOT NULL COMMENT '1=身份证正面 2=身份证反面 3=个人照片',
    file_url VARCHAR(500) NOT NULL COMMENT '文件URL',
    audit_status TINYINT DEFAULT 0 COMMENT '0=待审核 1=已通过 2=不通过',
    audit_remark VARCHAR(500) COMMENT '审核备注（不通过原因）',
    audited_at DATETIME COMMENT '审核时间',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id),
    INDEX idx_audit_status (audit_status)
);
```

#### 2. 题库

```sql
-- 题目分类表
CREATE TABLE question_category (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL COMMENT '分类名称',
    sort_order INT DEFAULT 0 COMMENT '排序',
    status TINYINT DEFAULT 1 COMMENT '1=启用 0=停用',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 题目表
CREATE TABLE question (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    question_no VARCHAR(32) UNIQUE COMMENT '题目编号，自动生成',
    content TEXT NOT NULL COMMENT '题目内容',
    media_url VARCHAR(500) COMMENT '媒体附件URL（图片/视频/GIF）',
    media_type TINYINT COMMENT '媒体类型 1=图片 2=视频 3=GIF',
    question_type TINYINT NOT NULL COMMENT '1=单选 2=多选 3=判断',
    category_id BIGINT COMMENT '分类ID',
    score DECIMAL(5,1) DEFAULT 1 COMMENT '分值',
    analysis TEXT COMMENT '知识点解析',
    status TINYINT DEFAULT 1 COMMENT '1=生效 0=失效',
    created_by VARCHAR(50) COMMENT '创建人',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_category (category_id),
    INDEX idx_status (status),
    INDEX idx_type (question_type)
);

-- 题目选项表
CREATE TABLE question_option (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    question_id BIGINT NOT NULL,
    option_label VARCHAR(10) NOT NULL COMMENT '选项标识 A/B/C/D',
    option_content TEXT NOT NULL COMMENT '选项内容',
    is_correct TINYINT DEFAULT 0 COMMENT '是否正确答案',
    sort_order INT DEFAULT 0,
    INDEX idx_question_id (question_id)
);
```

#### 3. 课程与考试

```sql
-- 课程表
CREATE TABLE course (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL COMMENT '课程名称',
    description TEXT COMMENT '课程描述',
    status TINYINT DEFAULT 1 COMMENT '1=启用 0=停用',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 考试/科目表（不直接绑定课程，通过 course_exam 多对多关联）
CREATE TABLE exam (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    UNIQUE KEY uk_exam_question (exam_id, question_id)

-- 随机抽题规则表（随机抽题模式，按分类比例抽题）
CREATE TABLE exam_random_rule (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    exam_id BIGINT NOT NULL,

    percentage DECIMAL(5,2) NOT NULL COMMENT '该分类占比（%），向下取整，差额从未配置分类随机补',
    INDEX idx_exam_id (exam_id)

### 状态机

#### 材料审核状态（user_material.audit_status）
```
未上传 → 待审核(0) → 已通过(1)
                    → 不通过(2) → 重新上传 → 待审核(0)
```

#### 报考材料审核状态（enrollment.audit_status）
```
待审核(0) → 已通过(1) → 可考试
          → 不通过(2) → 重新提交材料 → 待审核(0)
代报名：直接设为 已通过(1)
```

#### 报考考试状态（enrollment.exam_status）
```
未开始(0) → 考试中(1)  -- 任一科目有考试记录时
         → 已通过(2)  -- 该课程下所有科目全部通过时自动更新
         → 未通过(3)  -- 存在科目未通过且重考次数用完时
```
触发时机：每次考试交卷后，检查该课程下所有科目的通过状态，自动更新 exam_status。

#### 考试记录状态（exam_record.result）
```
开始考试 → 人脸核身 → 答题中 → 主动交卷(1) → 通过(1)/未通过(0)
                              → 超时自动交卷(2) → 通过(1)/未通过(0)
                              → 中途退出交卷(3) → 通过(1)/未通过(0)
                              → 作弊交卷(4) → 通过(1)/未通过(0)
```

```

question_category (1) ──── (N) question   一个分类有多道题
